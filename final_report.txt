\section{Introduction}
%% ============================================================

Retrieval-Augmented Generation (RAG) has become a standard approach for grounding LLM outputs in external evidence. Yet a critical question remains underexplored: \textit{when the model retrieves multiple documents, which ones actually matter for correct generation?}

This question is not merely academic. In high-stakes domains---medicine, law, finance---a retrieved document that confidently asserts a false fact can be far more dangerous than no document at all. If an attribution system labels such a document as ``important'' only because it shifts the model's output, it provides misleading guidance: important-sounding evidence that actively degrades reliability.

Existing attribution methods (ContextCite~\cite{cohen2024contextcite}, TracLLM~\cite{wang2025tracllm}, AttriBoT~\cite{dubey2025attribot}) define importance as \emph{causal contribution to the output}. A document that strongly shifts the model's response receives a high importance score---regardless of whether that shift improves or harms factual correctness. This is the core gap this project addresses: \textbf{attribution methods identify what influences the model's output, but fail to distinguish whether such influence improves or degrades factual correctness.}

We propose \textbf{LODO (Leave-One-Document-Out)}, a three-axis evaluation framework that measures, for each removed document: factual correctness drop, log-probability collapse of the baseline answer, and layer-wise representation drift inside the transformer. Together these axes expose whether a document's causal contribution is factually aligned---and reveal internal model fragility that surface accuracy metrics miss entirely.

We conduct all experiments on \textbf{en\_counter\_mid}, the counterfactual subset of the RGB benchmark~\cite{chen2024rgb}, in which retrieved documents confidently assert incorrect answers. This setting makes individual document contribution maximally observable: the model's correct output must actively resist the retrieved evidence, so removing a single document creates a measurable causal perturbation. We additionally run a \textbf{passage-number sweep} (passage\_num $\in \{3, 5, 7, 10\}$) to characterize how the number of retrieved documents affects causal importance and internal representation.


%% ============================================================
\section{Related Work}
%% ============================================================

\subsection{Attribution in RAG}

ContextCite~\cite{cohen2024contextcite} trains a sparse linear surrogate via LASSO to estimate how context ablation changes output probability. TracLLM~\cite{wang2025tracllm} handles long contexts via recursive partitioning and contribution-score denoising. AttriBoT~\cite{dubey2025attribot} approximates leave-one-out attribution efficiently using cached activations and proxy models. All three methods are evaluating \emph{influence}, not \emph{factual usefulness}. A document that confidently misleads the model would score as highly important under all three.

\subsection{Gap This Work Addresses}

This project introduces factual correctness as a first-class criterion in document importance: a document is important not merely if it causally affects the output, but if its causal contribution is \emph{aligned with correct generation}. We further extend the analysis into the model's internal representations to locate \emph{where} in the transformer this influence operates---a dimension entirely absent from prior attribution work.


%% ============================================================
\section{Dataset: Why en\_counter\_mid}
%% ============================================================

We use the \textbf{en\_counter\_mid} subset of RGB, which presents queries where retrieved documents assert an answer that contradicts the ground truth. This creates a hard counterfactual robustness setting: the model must resist the retrieved evidence to produce the correct answer.

\textbf{Why not en\_mid?} Initial experiments on the standard \texttt{en\_mid} subset showed near-ceiling accuracy (92--96\%), meaning the model answers correctly whether or not any single document is present. LODO is causally uninformative in this regime: factual degradation is effectively unobservable. Figure~\ref{fig:dataset_motivation} illustrates this contrast directly.

\begin{figure}[h]
    \centering
    \includegraphics[width=\linewidth]{plots/CF4_dataset_motivation.png}
    \caption{Accuracy on \texttt{en\_mid} vs.\ \texttt{en\_counter\_mid}. \texttt{en\_mid} is near-ceiling (92--96\%), making document removal a non-event. \texttt{en\_counter\_mid} places accuracy at 8--20\%, creating a setting where each document's presence is causally meaningful.}
    \label{fig:dataset_motivation}
\end{figure}

On \texttt{en\_counter\_mid}, average accuracy drops to 8--20\% depending on the number of retrieved documents. Whether the model arrives at the correct answer depends critically on which specific documents are present---the ideal setting for LODO to reveal causal document importance.


%% ============================================================
\section{LODO Mechanistic Evaluation}
%% ============================================================

\subsection{Setup}

For each query in \texttt{en\_counter\_mid}, we construct a full-context baseline and generate a baseline answer with Llama-3.1-8B-Instruct ($\tau=0.7$). We then remove each document in turn and measure three degradation axes relative to the baseline:

\begin{itemize}[noitemsep]
    \item $\Delta_{\text{fact}}$: binary drop in factual correctness of a newly generated answer.
    \item $\Delta_{\text{logprob}}$: reduction in log-likelihood of the original baseline answer under the ablated context (teacher forcing; fully deterministic, unaffected by temperature).
    \item $\Delta_{\text{L2}}$: L2 distance between mean hidden states of baseline and ablated runs per transformer layer.
\end{itemize}

A document is classified causally important if $\Delta_{\text{fact}} > 0$ or $\Delta_{\text{logprob}} < -2.0$.

\subsection{Passage-Number Sweep}

We run LODO with passage\_num $\in \{3, 5, 7, 10\}$ on 10 queries from \texttt{en\_counter\_mid}, holding the query set and random seed fixed. For each passage\_num, documents are sampled with at least one positive document; the remainder are negative (noise) documents.

\textbf{Hypothesis.} With fewer documents, each document carries more informational weight, so removing any single one should produce larger degradation and higher causal importance rates. With more documents, redundancy increases and the model can compensate, so importance becomes sparser.

\subsection{Key Findings from the Sweep}

\textbf{Finding 1: Two distinct regimes separated at $n=3$.}
At passage\_num=3, 50\% of ablations are causally important with a mean logprob degradation of $-13.06$---nearly four times larger than at $n=5$ ($-3.39$). However, the baseline fact score at $n=3$ is 0\% across all queries: the model never answers correctly with only 3 counterfactual documents. This reveals a \emph{context starvation} regime: with too few documents, the context window provides insufficient grounding and the model defaults to parametric knowledge that may be incorrect. In this regime, any document removal causes collapse not because the document is factually useful, but because the model has too little context to operate on. At $n \geq 5$, the model enters a \emph{redundancy regime} where baseline accuracy becomes non-zero and causal importance stabilizes.

\textbf{Finding 2: Redundancy effect---drift and logprob degradation decrease monotonically.}
Mean L2 drift at layer 32 decreases from 18.08 ($n=3$) to 8.45 ($n=10$), and mean logprob degradation decreases from $-13.06$ to $-2.15$. Each added document provides an additional compensation pathway: when one document is removed, others carry overlapping information and the model's internal state is less disrupted. This empirically validates the redundancy hypothesis.

\textbf{Finding 3: Factual degradation increases with $n$.}
Counterintuitively, the number of $\Delta_{\text{fact}} > 0$ cases \emph{increases} with passage count (0, 1, 5, 10). This is because factual degradation requires the baseline to be correct first---only at $n \geq 5$ does the model occasionally answer correctly, creating conditions where a single document removal can flip correctness. At $n=10$, 10 out of 90 ablations cause factual degradation, with 2 queries having a correct baseline. This means that as the retrieved context becomes richer and more sufficient, LODO's factual degradation axis becomes progressively more informative.

\textbf{Finding 4: Positive and negative documents are equally likely to be causally important at small $n$.}
Figure~\ref{fig:pos_neg} shows that at $n=3$, positive (ground-truth-supporting) and negative (counterfactual) documents have similar causal importance rates. At $n=10$, positive documents dominate importance ($14/23$ causally important ablations). This is consistent with the hypothesis that at larger $n$, the model's correct outputs depend more on the positive documents that resist the counterfactual narrative, while negative documents become mutually redundant.

\subsection{Sparsity and Collapse Divergence}

In the redundancy regime ($n \geq 5$), the majority of ablations produce no measurable degradation. At $n=5$, only 22\% of ablations are causally important; at $n=10$, 25.6\% despite having twice as many ablations total. This sparsity has a direct implication: most retrieved documents are \emph{replaceable}. The model can compensate for their absence via parametric knowledge or remaining documents.

\textbf{Collapse divergence} remains prominent across all passage\_num conditions: severe logprob collapse ($\Delta_{\text{logprob}} < -2.0$) frequently co-occurs with no factual degradation ($\Delta_{\text{fact}} = 0$). At $n=5$, all 11 causally important cases by logprob involve 0 factual degradation (the one factual degradation case is a separate ablation). This confirms the core finding: the model exhibits internal fragility that surface accuracy metrics cannot detect, and this pattern persists regardless of how many documents are retrieved.

\subsection{Integration Horizon: Where Evidence Is Processed}

Layer-wise drift trajectories reveal a consistent structural pattern across all passage\_num conditions: drift is near-zero through layers 0--17, then rises sharply from approximately \textbf{layer 18--20}. We term this the \textbf{Integration Horizon}---the transition point where the transformer shifts from syntactic/structural encoding to factual grounding in retrieved context.

Token-wise logprob analysis (Figure~\ref{fig:tokenwise}) shows that document influence is further localized within the output: only 1--2 factual tokens (named entities, numbers) experience significant logprob collapse. The rest of the generated answer---grammar, prepositions, sentence structure---remains unaffected by document removal. This token-level sparsity parallels the document-level sparsity: causal importance is concentrated in a small fraction of both documents and output tokens.

\section{Discussion}
%% ============================================================

\subsection{Answering the Reviewer Comments}

\textbf{Real-world danger of the gap.} A medical RAG system retrieving a document with an incorrect drug dosage may show collapse divergence: massive internal confidence loss and representation drift, but correct surface output on most sampling runs. Under a different temperature setting or rephrased query, the surface error becomes observable---but by then the system has already been deployed. LODO's three-axis framework provides an early-warning signal that surface accuracy evaluation cannot.

\textbf{Scalability.} Standard LODO runs in $O(n)$ ablations per query ($n$ = passage\_num). The passage-number sweep directly addresses whether this scales meaningfully: at passage\_num=3 each ablation is a large fraction of total context; at passage\_num=10 the model has many more compensation paths. The sweep characterizes this redundancy curve empirically. Leave-$k$-out for $k>1$ grows as $O(n^k)$ and is left for future work; our results suggest single-document ablation is sufficient to identify individually critical documents.

\textbf{Positional encoding confound.} Standard LODO removes document $i$, shifting all subsequent document positions and changing their RoPE encodings. We address this with a \textbf{position-controlled ablation} design (implemented in \texttt{run\_lodo\_position\_controlled.py}): replacing removed documents with length-proportional neutral placeholders (\texttt{[DOCUMENT REMOVED]}) keeps document count, approximate sequence length, and positional structure identical. The positional encoding artifact is then:
\[
\Delta_{\text{PE}} = \Delta_{\text{L2}}^{\text{standard}} - \Delta_{\text{L2}}^{\text{controlled}}
\]
Running this controlled experiment is a direct next step; the implementation is ready.

\textbf{Temperature confound.} $\Delta_{\text{logprob}}$ and $\Delta_{\text{L2}}$ are computed via teacher forcing over the fixed baseline answer---fully deterministic given the input, unaffected by temperature. Only $\Delta_{\text{fact}}$ involves sampling; we treat it as binary to minimize variance.

\subsection{Passage Number and Document Importance}

The passage-number sweep allows us to characterize the \emph{redundancy curve}: how causal importance rate changes as more documents are added. This has direct practical implications for RAG system design---if importance is already sparse at $n=5$, adding more documents may not improve factual grounding but increases the risk of introducing a causally harmful document that the model cannot resist.


%% ============================================================
\section{Further Plans}
%% ============================================================

\begin{enumerate}[noitemsep]
    \item \textbf{Run position-controlled LODO}: Execute \texttt{run\_lodo\_position\_controlled.py} to quantify $\Delta_{\text{PE}}$ and validate that the integration horizon is a semantic---not positional---effect.
    \item \textbf{Distributional importance threshold}: Replace the heuristic $\Delta_{\text{logprob}} < -2.0$ with a per-query distributional criterion ($> 2\sigma$ below the query mean), making importance classification adaptive to query difficulty.
    \item \textbf{Predictive signals}: Investigate whether mid-layer drift ($\sim$layer 20) can serve as a real-time signal to flag uncertain generation before decoding completes.
\end{enumerate}
